# History Page Redesign Plan

## Goal
Redesign `/history` page from a basic list layout to a polished grid card layout with date grouping, search/filter, hover video preview, Watch Later / Add to Queue actions, and a horizontal-scroll "Continue Watching" section.

## Current State
- **File**: `frontend/src/app/history/page.tsx` (221 lines, single "use client" component)
- **Layout**: Simple vertical list of horizontal cards (thumbnail left + details right)
- **Sections**: "Continue Watching" + "All History" — both as simple `space-y-3` stacks
- **No**: date grouping, search/filter, hover preview, Watch Later/Queue actions, skeleton loading, grid layout

## Target Design

### 1. Page Structure
```
[Sidebar] [Navbar]
<main>
  ├── Header (title + search bar + clear-all button)
  ├── Continue Watching (horizontal scroll row with arrows)
  └── Date-Grouped Grid Sections
        ├── Today
        ├── Yesterday
        ├── This Week
        └── Older
</main>
```

### 2. New Components to Create

#### `src/components/history/HistoryPageClient.tsx`
Extract the full client logic from the page. The `page.tsx` becomes a thin wrapper.

#### `src/components/history/HistoryPageSkeleton.tsx`
- Skeleton loading state matching the grid layout

### 3. Date Grouping Logic
Add a `groupByDate()` utility function in the page component:
```ts
type DateGroup = "Today" | "Yesterday" | "This Week" | "This Month" | "Older";
function groupByDate(items: WatchHistoryItem[]): { label: DateGroup; items: WatchHistoryItem[] }[]
```
- Compares `watchedAt` (ms timestamp) to `Date.now()`
- Today = same calendar day
- Yesterday = 1 day ago
- This Week = 2-7 days ago
- Older = > 7 days ago

### 4. Search/Filter Logic
- Client-side filtering after data is fetched
- `searchQuery` state (debounced)
- Filters: `item.title.toLowerCase().includes(query) || item.channelName.toLowerCase().includes(query)`
- When search is active, disable date grouping and show flat filtered results with a result count
- Clear button (X icon) in search bar when query is non-empty

### 6. Design Specifications

#### Color & Styling (matching app conventions)
- Background: `bg-[#09090b]` (matching home page, current uses `#0f0f0f` — update to match)
- Cards: `bg-white/[0.02] ring-1 ring-white/[0.06] hover:bg-white/[0.04] hover:ring-white/[0.1]`
- Progress bar: `bg-gradient-to-r from-red-500 to-red-400` with glow
- Hover play overlay: `bg-white/90` circle with `Play` icon (fill black)
- Section headers: Use `SectionHeader` component with appropriate accent colors
  - Continue Watching: `from-red-500 to-orange-500`
  - Today: `from-violet-500 to-indigo-500`
  - Yesterday: `from-blue-500 to-cyan-500`
  - This Week: `from-emerald-500 to-teal-500`
  - Older: `from-amber-500 to-orange-500`
- Search bar: `rounded-full bg-secondary/50 border-white/10` (matches navbar)
- Clear all button: Ghost variant with `Trash2` icon
- Glassmorphism panels for header area: `bg-white/[0.02] backdrop-blur-sm ring-1 ring-white/[0.06]`

#### Responsive Grid
- 1 col on mobile (`grid-cols-1`)
- 2 cols on `sm` (`sm:grid-cols-2`)
- 3 cols on `lg` (`lg:grid-cols-3`)
- 4 cols on `xl` (`xl:grid-cols-4`)

#### Max Width
- `max-w-[1920px]` (matching home page, current is `max-w-5xl` — widen)

### 7. Implementation Steps

1. **Create `HistoryVideoCard`** — the core card component with HoverVideoPlayer, progress bar, hover actions (Watch Later, Add to Queue, Remove), metadata
2. **Create `HistoryGridSection`** — section wrapper with SectionHeader + responsive grid
3. **Create `ContinueWatchingRow`** — horizontal scroll row with arrow navigation (port from home page pattern)
4. **Create `HistorySearchBar`** — search input with debounce + clear
5. **Create `HistoryPageSkeleton`** — skeleton loading state
6. **Create `HistoryPageClient`** — main component wiring: data fetching (useQuery), search state, date grouping, rendering all sections
7. **Update `page.tsx`** — thin wrapper with auth guard, renders `HistoryPageClient`
8. **Delete inline `HistoryCard`** and old code from `page.tsx`

### 8. Edge Cases
- **Unauthenticated state**: Keep existing sign-in prompt (no changes needed)
- **Empty state**: Keep existing "No videos here" empty state
- **Empty search results**: Show "No results found for '{query}'" with clear search button
- **Continue Watching empty**: Don't render the section (return null)
- **Pagination**: Backend supports `page`/`limit` — for now, load first 50 items. Add "Load More" button at bottom if `total > 50`. Future: infinite scroll.
- **Items that appear in both Continue Watching and All History**: Keep duplicate — Continue Watching links to `/watch/{id}?t={progress}`, All History links to `/watch/{id}`

### 9. No Backend Changes
All backend routes are already sufficient. The frontend already has:
- `api.user.getHistory()` — returns `{ items: WatchHistoryItem[] }`
- `api.user.removeFromHistory(videoId)` — deletes single entry
- `api.user.clearHistory()` — clears all
- Backend supports pagination (`page`, `limit` query params)
